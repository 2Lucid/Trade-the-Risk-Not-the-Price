# Interactive Lab — `site/`

A self-contained, static interactive companion to the research. It **recomputes
the volatility-managed strategy live in the browser** (the engine in `app.js`
mirrors `src/strategy.py` exactly), so you can choose the asset, the volatility
forecaster, the trend overlay, the leverage cap and the trading cost, and watch
the Sharpe ratio, return, drawdown and equity curve update instantly.

## Run it

Just open `index.html` — no server, no build step (data is inlined in `data.js`).

```bash
open index.html          # macOS
# or double-click index.html
```

## Files

| File | What |
|---|---|
| `index.html` | structure + sections |
| `style.css`  | the design system |
| `app.js`     | strategy recompute engine + Plotly charts |
| `data.js`    | inlined results (`window.VMP_DATA`), generated by the pipeline |

`data.js` is regenerated from the pipeline with:

```bash
python -m src.export_site
```

## Deploy (optional)

Any static host works. For example, **GitHub Pages**: commit the repo, enable
Pages, and point it at the repo root or `/site`. Then set `\siteurl` in
`paper/main.tex` to the published URL and recompile, so the PDF links to your
live site. The `paper/main.pdf` link in the footer assumes the compiled paper is
served alongside the site (copy `paper/main.pdf` next to `index.html`, or adjust
the link).

> Requires an internet connection to load Plotly.js from its CDN.
> Research/education only — not investment advice.
